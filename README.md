AutoAbiba
AutoAbiba — це легка утиліта для Windows (на базі WPF/.NET), яка автоматично керує фоновими утилітами або стороннім софтом для ваших улюблених ігор. Програма моніторить запущені процеси та запускає відповідний софт, коли гра відкривається, і автоматично закриває його, коли ви виходите з гри.

🚀 Як це працює
Додаток працює у фоновому режимі та кожні 3 секунди перевіряє список процесів:

Гра запустилася? ➡️ Автоматично запускає відповідну програму з кореневої папки.

Гра закрилася? ➡️ Автоматично завершує (Kill) процес супутньої програми, щоб вона не споживала ресурси.

🛠️ Налаштування (Configuration)
Щоб додати свої ігри та софт, відредагуйте словник gamesConfig у файлі App.xaml.cs:

C#
var gamesConfig = new Dictionary<string, string>
{
    // { "назва_процесу_гри", "назва_файлу_софту.exe" }
    { "cs2", "cs2_companion.exe" },
    { "PioneerGame-e", "stalcraft_overlay.exe" },
    { "RustClient", "rust_helper.exe" }
};
⚠️ Важливо: Файли супутнього софту (наприклад, cs2_companion.exe) мають лежати в тій самій папці, де знаходиться і сам AutoAbiba.exe (поруч із виконуваним файлом).

📋 Вимоги (Requirements)
.NET Core 6.0 / 7.0 / 8.0 (або новіша версія)

ОС Windows (оскільки використовуються WPF та Process.Start)

🛠️ Архітектурні особливості

Асинхронність: Моніторинг запущений через Task.Run(), що дозволяє не блокувати головний UI-потік (UI thread) додатка WPF.

Ресурсоефективність: Використання Thread.Sleep(3000) гарантує, що нескінченний цикл не буде навантажувати процесор на 100%.

Безпека: Усі операції з процесами загорнуті в блоки try-catch для запобігання падінню додатка через брак прав доступу (наприклад, якщо гра запущена від імені Адміністратора).
